#+TITLE: ChromSex Integration
#+AUTHOR: Rohit Goswami

* Integrating with ChromSex

ChromSex is an R package for analyzing sex chromosome evolution in
salmonid fish. It consumes RADSex output files.

** Drop-in replacement

rsx-rs produces byte-identical output to C++ RADSex when groups are
specified explicitly (=-G M,F=). The existing ChromSex R functions
(=load_radsex_distribution=, =load_radsex_alignment=, =extract_signif_markers=)
work without any changes.

** Adding to ChromSex pixi.toml

The =build-rsx= task in ChromSex's =pixi.toml= builds rsx-rs from source:

#+begin_src toml
build-rsx = """bash -c 'if ! command -v rsx >/dev/null 2>&1; then \
  cd /tmp && git clone https://github.com/HaoZeke/rsx-rs.git && \
  cd rsx-rs && cargo build --release -j4 && \
  cp target/release/rsx ~/.local/bin/rsx; fi'"""
#+end_src

** Running from ChromSex

Use =system()= or =processx::run()= to call radsex from R:

#+begin_src R
system2("rsx", c(
  "distrib",
  "-t", "data/radsex/BT/markers_table.tsv",
  "-p", "data/popmaps/popmap_BT_sex_hestvatn.tsv",
  "-o", "data/radsex/BT/hestvatn_BT_distribution.tsv",
  "-d", "5", "-G", "M,F"
))
#+end_src

** Future: direct R FFI

The C API (=librsx_core.so=) will enable direct =.Call()= from R,
eliminating intermediate file I/O. See the
[[https://github.com/HaoZeke/rsx-rs/issues][issue tracker]] for progress.
