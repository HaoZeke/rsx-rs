* Quick Start

** Installation

*** From source (recommended)

#+begin_src bash
git clone https://github.com/HaoZeke/rsx-rs.git
cd rsx-rs
cargo build --release
# Binary at target/release/rsx
#+end_src

*** From GitHub releases

Pre-built binaries are available for Linux (x86_64, aarch64) and macOS
(x86_64, arm64) from the
[[https://github.com/HaoZeke/rsx-rs/releases][Releases page]]. On Windows,
build from source without the =map= feature (minimap2 is not available there).

*** Via pixi

#+begin_src bash
cd rsx-rs
pixi run build
#+end_src

** Example workflow

Given demultiplexed RAD-seq reads in =reads/= and a population map:

#+begin_example
# popmap.tsv
ind1    M
ind2    M
ind3    F
ind4    F
#+end_example

*** Step 1: Build markers table

#+begin_src bash
rsx process -i reads/ -o markers.tsv -T 4 -d 5
#+end_src

*** Step 2: Check marker frequencies

#+begin_src bash
rsx freq -t markers.tsv -o freq.tsv -d 5
#+end_src

*** Step 3: Compute sex-bias distribution

#+begin_src bash
rsx distrib -t markers.tsv -p popmap.tsv -o distrib.tsv -d 5 -G M,F
#+end_src

*** Step 4: Extract significant markers

#+begin_src bash
rsx signif -t markers.tsv -p popmap.tsv -o signif.tsv -d 5 -G M,F
#+end_src

*** Step 5: Map to reference genome

#+begin_src bash
rsx map -t markers.tsv -p popmap.tsv -g genome.fa -o aligned.tsv -d 5 -G M,F
#+end_src

*** Step 6: Merge multiple tables

#+begin_src bash
rsx merge -o combined.tsv pop1_markers.tsv pop2_markers.tsv pop3_markers.tsv
#+end_src

Uses bounded-memory external sort (~500MB) for arbitrarily large datasets.

*** Step 7: Streaming PCA

#+begin_src bash
rsx pca -t combined.tsv -o pca_results/ -d 5 -r 10
#+end_src

Produces eigenvalues, loadings, and summary in the output directory.
PC1 typically separates males and females for sex-linked markers.

** Output format

All outputs are tab-separated with an optional =#source:= comment line.
The format is identical to the original C++ RADSex tool, so existing
R scripts work without modification.

** Memory guarantees

All commands operate in bounded memory regardless of input size:

| Command         | Memory                |
|-----------------+-----------------------|
| distrib, freq   | O(n_individuals)      |
| signif, subset  | O(n_individuals)      |
| map             | O(genome_index)       |
| depth (small)   | O(n_markers * n_ind)  |
| depth (> 2GB)   | O(buffer_size)        |
| merge           | O(buffer_size)        |
| pca             | O(n_individuals^2)    |

For 200 individuals and 75M markers, typical peak memory is < 500MB
(except map which loads the minimap2 genome index).

** Python bindings (high-level API)

Install with =pip install pyrsx= (or =pixi run -e python build-python= from source).

#+begin_src python
import pyrsx

# Process reads → marker depth table (Arrow-backed under the hood)
pyrsx.process("reads/", "markers.tsv", threads=8, min_depth=5)

# Distribution + significance with Bayesian evidence
pyrsx.distrib("markers.tsv", "popmap.tsv", "distrib.tsv", groups=["M", "F"])
pyrsx.signif("markers.tsv", "popmap.tsv", "signif.tsv",
             groups=["M", "F"], test="fisher", correction="fdr", bayes=True)

# High-level ergonomic objects + narwhals / plotting
tbl = pyrsx.MarkerTable.from_path("markers.tsv")
print("n_markers:", len(tbl))

# Streaming PCA (Tucker mode-2) for sex signal QC
pyrsx.pca("markers.tsv", "pca_out/", n_components=5)

# Merge multiple tables (bounded memory)
pyrsx.merge(["run1.tsv", "run2.tsv"], "merged.tsv")
#+end_src

See the Python README for the full surface (including =from_dataframe=, =to_arrow=, custom triage, etc.).

** Reproducing the paper

Every benchmark, figure, and biological result reported for rsx is
reproducible from a single deposited archive. The Zenodo deposit
[[https://doi.org/10.5281/zenodo.20531539][doi:10.5281/zenodo.20531539]]
bundles the pinned workflow (=v0.2.3=), the downloaded literature inputs, the
result tables, and a one-command pixi + Snakemake pipeline:

#+begin_src bash
# from the extracted reproducibility archive
pixi install
pixi run bench        # regenerates results/ and results/figures/
#+end_src

The archive clones rsx at the pinned tag, builds it alongside the C++ RADSex
v1.2.0 reference, and regenerates the four-panel literature benchmark (the
8.38x geometric-mean speedup across 56 paired timings), the Bayesian evidence
grades, and the sex-linked marker calls. Timings scale with the host
hardware; the biological results (marker counts and evidence grades) do not.
