* Literature-Derived Benchmark Runs

This workflow downloads the public datasets used by the RADSex literature
workflow and runs =rsx= on those FASTQ files. The synthetic data in
=benchmarks/data/= remains a small, tracked performance and packaging check;
literature-derived benchmarks are the route for claims about behavior on
published RAD-seq sex determination datasets.

** Primary sources

- RADSex paper workflow:
  https://github.com/RomainFeron/paper-sexdetermination-radsex
- RADSex publication DOI: https://doi.org/10.1111/1755-0998.13360
- RADSex medaka example:
  https://radsex.readthedocs.io/en/docs/example.html
- Medaka source publication DOI:
  https://doi.org/10.1534/genetics.114.169284

** Inputs

The dataset manifest is tracked at
=benchmarks/literature_datasets.tsv=. It mirrors the RADSex paper workflow's
=data/info.tsv= source list:

- 15 fish datasets under BioProject =PRJNA548074= from the RADSex paper
  workflow.
- =oryzias_latipes= under BioProject =PRJNA253959= from the RADSex
  medaka example and Wilson et al. 2014.
- Genome links for datasets where the RADSex workflow specifies mapping
  inputs.

Validate the source list with:

#+begin_src bash
pixi run -e benchmark check-literature-manifest
#+end_src

** Benchmark commands

For the RADSex paper panel, run the command set used by the published
workflow across minimum depths 1, 2, 5, and 10:

#+begin_src bash
rsx process
rsx distrib
rsx signif
rsx freq
rsx depth
#+end_src

The medaka example additionally provides a published target for =map= and
=subset= with the example genome and medaka-specific thresholds from the
RADSex workflow configuration.

** Execution

Run the public medaka benchmark:

#+begin_src bash
pixi run -e benchmark run-literature-medaka
#+end_src

Run the full literature panel:

#+begin_src bash
pixi run -e benchmark run-literature-benchmarks
#+end_src

Generate Bayesian evidence summaries from completed =distrib= outputs:

#+begin_src bash
pixi run -e benchmark analyze-literature-bayes
#+end_src

Run same-input C++ RADSex vs rsx timing comparisons after building the C++
binary:

#+begin_src bash
pixi run -e benchmark compare-literature-radsex --radsex /path/to/radsex/bin/radsex
#+end_src

On a Slurm cluster, use the per-dataset array script so the comparison stage
measures compute time from prepared FASTQ/marker-table inputs rather than
download throughput:

#+begin_src bash
sbatch --export=ALL,RSX_REPO=$PWD benchmarks/slurm/rsx_release_build.sbatch
sbatch --export=ALL,RSX_REPO=$PWD benchmarks/slurm/radsex_cpp_build.sbatch
sbatch --export=ALL,RSX_REPO=$PWD benchmarks/slurm/literature_prepare_dataset.sbatch
sbatch --export=ALL,RSX_REPO=$PWD,RADSEX_BIN=/path/to/radsex \
  benchmarks/slurm/literature_radsex_compare.sbatch
pixi run -e benchmark merge-literature-slurm-results
#+end_src

Run Python-binding feature evidence on a published RADSex workflow dataset:

#+begin_src bash
pixi run -e paper run-literature-bindings
#+end_src

Regenerate paper figures and summary TSVs:

#+begin_src bash
pixi run -e benchmark plot-literature-benchmarks
#+end_src

=benchmarks/literature-workdir/=, which is intentionally ignored by git. The
tracked measured outputs are:

** Biological interpretation anchors

The four executed datasets deliberately include both strict RADSex-positive
and strict RADSex-null cases from Feron et al. 2021:

| Dataset                  | Source-paper sex-determination result | Source-paper marker count |
|--------------------------+----------------------------------------+---------------------------|
| =danio_albolineatus=     | No SD identified                       | 0 at d=10                 |
| =notothenia_rossii=      | No SD identified                       | 0 at d=10                 |
| =plecoglossus_altivelis= | XX/XY, previously described            | 47 at d=10                |
| =tinca_tinca=            | XX/XY, previously undescribed          | 6 at d=10                 |

This split keeps the performance benchmark tied to biological use cases:
=plecoglossus_altivelis= and =tinca_tinca= are positive controls for
recovering male-heterogametic signal, while =danio_albolineatus= and
=notothenia_rossii= test throughput and candidate ranking on large null
panels where posterior-ranked evidence must not be reported as confirmed
sex-determining loci.

** Speed comparison summary

The tracked =docs/figures/literature_speedup_summary.tsv= reports 56 paired
C++ RADSex vs rsx timings across four datasets, command groups, and minimum
depths. Rust wins all 56 pairs. Geometric-mean speedups are:

| Command family | Geometric mean speedup |
|----------------+------------------------|
| =process=      |                  1.83x |
| =depth=        |                  2.99x |
| =freq=         |                  4.57x |
| =distrib=      |                  5.81x |
| =signif=       |                  1.36x |

** Reporting split

- Synthetic suite: exact tracked timing CSV, figures, packaging smoke test,
  and CI-sized regression data.
- Literature suite: downloaded public biological datasets, measured command
  timings, marker counts, and output counts from the RADSex workflow
  parameterization.

The paper should identify which result table comes from which suite.
Synthetic timings must not be presented as the biological benchmark
corpus.
